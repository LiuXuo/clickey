# Clickey 迭代总结

## 总体目标

- 通过“分层网格 + 键盘定位”快速定位鼠标点击位置。
- 结合遮罩网格、回退/取消、以及不同精度策略提升可用性。

## 使用背景与灵感

- 使用设备为直列 ZMK 固件分体键盘，主键第 0 层为 `qwert   yuiop` / `asdfg   hjkl;` / `zxcvb   nm,./`，所以定位键始终围绕这套主键布局，尽量贴近物理方位和手指落点。
- 灵感来自 Vim / Vimium / Vimac / Vimium C / Win-vind / Warpd等产品；迭代中发现与 mouseless 的理念高度接近，纯属巧合。
- 触发采用修饰键 + `;`，分号位于右手小指候选键位，便于高频激活且冲突概率较低。
- `Esc` / `Backspace` 的交互符合直觉；`Space` 配合交叉对角线可随时跳过多余层级。

## 版本演进

### v1.0

- 结构为 3 层：9x9 双键（`wer/sdf/xcv` 行键 + `uio/jkl/m,.` 列键）+ 两层 3x3 单键。
- 以验证 AHK 可行性为主，使用左右手食指/中指/无名指交叉定位，屏幕划分 6561 区域。
- 敲击节奏清晰、利于肌肉记忆且对小拇指友好，但四键敲击次数偏多。
- 支持 DPI 缩放与多显示器虚拟屏幕范围，并提供 `Ctrl+Alt+D` 调试信息。

### v1.1

- 为减少敲击次数并更充分利用按键，改为 2 层结构，使用左手三行 + 右手两行做 5x5 组合，覆盖 25x25。
- 行列布局尽量贴合键位物理方位；未采用六行五列，避免层级递进后网格比例变形，且 `nm,./` 行的 `,` 与 `.` 难区分。
- 屏幕划分 15625 区域，比 v1.0 更精细且敲击更少；定位流程为“先列后行”。

### v2.0

- 参考 mouseless 官网视频中的键位方案，追求更少离开候选键位的手指移动。
- 双键层升级为 30x10，单键层为 3x10；仍沿用“先列后行”。
- 尝试彩色网格后发现视觉干扰明显，最终放弃。

### v2.1

- 双键层提升到 30x30，行列使用第 0 层全部按键；单键层收敛为左手主键位（`qwert/asdfg/zxcvb`）。
- 目标是进一步提高密度，同时让单键层更贴近主键手感；屏幕划分 13500 区域。

### v2.2

- 单键层加入“主键 + 替代键”双映射（`qwert/asdfg/zxcvb` 与 `yuiop/hjkl;/nm,./`），实际使用中更贴近“哪只手空就用哪只手”。
- 为降低噪声，单键层不再绘制字母标签，仅保留网格。

### v3.0

- 不再盲目追求高密度：双键层行列减半并错开，可先行后列或先列后行，常用节奏是“左-右-左”交叉敲击，效率较同手敲击高。
- 大部分场景已足够；为补足特殊精度需求，移除 v2.2 的单层双映射，改在右手区加入三套步长的类 Vim 方向微调（10px/5px/1px）。

### v3.1

- 实践后认为“先行或先列”意义不大且限制未来布局，回归固定顺序（先列后行）。
- 三套步长微调冗余且占右手键区，收敛为方向键 5px 微调。
- 加入 `Tab` 多显示器切换，DPI 处理改为几何按原始像素绘制、字体按 DPI 缩放，避免多屏 DPI 差导致遮罩偏移。
- 提高遮罩透明度并开启 `-DPIScale` 与加粗字体。
